# AI Workspace Pro (Local Browser LLM)

Минималистичное веб-приложение для локального анализа текста на базе языковой модели **Qwen-1.5-0.5B-Chat**, работающей прямо в браузере. Никаких сторонних API-ключей, серверов и отправки ваших данных во внешние сети — все вычисления происходят на стороне клиента (On-Device AI) с использованием технологий WebAssembly (WASM).

Интерфейс спроектирован в чистом, контрастном и строгом стиле (вдохновленном эстетикой Vercel) — белый фон, глубокие черные акценты, тонкие границы и аккуратные закругления.

---

## ⚡️ Ключевой функционал

*   **🤖 Детектор ИИ-контента (AI Detector):**
    *   Анализирует вероятность генерации текста нейросетями.
    *   Рассчитывает метрику **Burstiness** (вариативность длины предложений и структуры текста).
    *   Определяет **уникальность словарного запаса** (Linguistic Diversity).
    *   Выводит структурированное экспертное заключение от локальной LLM.
*   **🎭 Анализ тональности текста (Sentiment & Toxicity):**
    *   Оценка эмоционального окраса (Позитивный / Негативный / Нейтральный).
    *   Определение уровня токсичности и кликбейта в процентном соотношении.
*   **📝 Краткая выжимка (TL;DR Summary):**
    *   Автоматическое выделение 3-х ключевых тезисов из объемного текста.
    *   Подсчет входящих предложений и символов.
*   **📎 Поддержка локальных файлов:**
    *   Возможность загружать текстовые файлы (`.txt`) напрямую для последующего анализа.
*   **💾 Локальное сохранение истории:**
    *   Интеграция с `localStorage` сохраняет текущую историю чата даже при перезагрузке страницы.

## 🛠️ Стек технологий и библиотеки

*   **Transformers.js (`@xenova/transformers`):** Библиотека для запуска моделей Hugging Face прямо в веб-приложении. Использует ONNX Runtime Web под капотом.
*   **Модель:** `Xenova/Qwen1.5-0.5B-Chat` в квантованном формате **Q4** (4-битное сжатие), что позволяет загружать модель весом всего около ~350 MB прямо в оперативную память браузера без потери критического качества генерации.
*   **WebAssembly (WASM):** Обеспечивает высокую производительность выполнения нейросетевых вычислений на процессоре клиента (CPU) в один поток для стабильности работы.
*   **Frontend:** Чистый HTML5, CSS3 (современные CSS-переменные, Flexbox, CSS Grid) и асинхронный ES6+ JavaScript. Без тяжелых фреймворков для максимальной скорости отрисовки.

## 🚀 Как запустить проект локально

Поскольку приложение работает на базе ES-модулей и загружает внешние файлы ONNX-модели, запуск просто двойным кликом по файлу `index.html` может быть заблокирован политикой CORS в браузере.

Для запуска требуется любой локальный статический сервер.

### Способ 1: С помощью Live Server (VS Code)
1. Установите расширение **Live Server** в VS Code.
2. Откройте папку с файлом `index.html`.
3. Нажмите кнопку **Go Live** в правом нижнем углу редактора.

### Способ 2: С помощью Python (если установлен)
Запустите команду в терминале внутри папки с проектом:
```bash
python -m http.server 8000
